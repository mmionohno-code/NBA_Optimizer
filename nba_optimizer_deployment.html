Portfolio Deployment Strategy

NBA
OPTIMIZER

A phased roadmap to take your project from local Python scripts to a published, recruiter-ready portfolio piece.

01
Clean Up & Document Your Codebase
Est. 2–3 days · Do first
START HERE
  • Rename files consistently — use snake_case (e.g., nba_data_pull.py, optimizer.py, clustering.py)
  • Add a docstring to the top of every Python file explaining what it does in 2–3 sentences
  • Create a requirements.txt by running pip freeze > requirements.txt in your project folder
  • Create a .gitignore file to exclude __pycache__/, .env, and large CSV files from being committed
  • Move all raw data CSVs into a /data subfolder; move scripts into /src or keep flat — just be consistent
  • Add inline comments to your most complex functions (K-Means clustering, MILP optimizer, PCA weighting)
Why this matters: Recruiters and data science hiring managers will look at your actual code on GitHub. Clean, commented code signals professionalism even more than the project idea itself.
02
Build a Standout GitHub Repo
Est. 1–2 days · Your public face
HIGH IMPACT
  • Write a compelling README.md — this is the most important file in the repo
    See structure below ↓
  • Add a repo description and topic tags on GitHub (e.g., nba, python, sports-analytics, optimization, machine-learning)
  • Pin this repo to your GitHub profile so it's the first thing visitors see
  • Include 1–2 sample output screenshots or a results table in the README (even a small PNG of cluster results)
  • Add a LICENSE file (MIT is standard for portfolio work)
# Ideal README.md structure # NBA Roster Optimizer Brief 2-sentence hook: what it does + why it matters ## Methodology - 3-season data (2021–24) with recency weighting (20/35/45%) - K-Means archetype clustering (position-agnostic) - PCA-derived stat weights + MinMaxScaler normalization - MILP optimization across 3 salary cap scenarios - VORPD metric for value-over-replacement ## Key Results [Screenshot or table of your best output here] ## How to Run pip install -r requirements.txt python src/optimizer.py ## Skills Demonstrated Python · pandas · scikit-learn · PuLP · nba_api
03
Resolve the Salary Data Gap
Est. 1–3 days · Critical blocker
BLOCKER
Option A — Best
basketball-reference.com via Stathead or manual download
Option B — Easiest
Spotrac salary tables (CSV export or manual copy)
Option C — API
balldontlie.io (free API, has salary data for recent seasons)
Option D — Fallback
Kaggle NBA salary datasets (search "NBA salaries 2022 2023 2024")
  • Try balldontlie.io first — free key, clean JSON, matches player names from nba_api
  • If using Kaggle, verify salary year alignment with your three seasons before merging
  • Standardize player names (handle "LeBron James" vs "Lebron James" mismatches) before merging on player ID or name
  • Document your salary data source in the README so reviewers know it's intentional, not missing
Portfolio framing tip: Even if salary data is imperfect, document the limitation clearly. Recruiters respect analytical honesty — it shows research maturity.
04
Create a Showcase Notebook or PDF Report
Est. 3–5 days · The hero artifact
HIGH IMPACT
  • Create a Jupyter Notebook (analysis_showcase.ipynb) that walks through your methodology end-to-end with narrative text between cells
  • Include a cluster visualization — 2D scatter plot of player archetypes using PCA components (matplotlib or seaborn)
  • Include an optimized roster table showing players selected per cap scenario with VORPD scores and salaries
  • Add a correlation heatmap from your three-framework analysis
  • Export the notebook as a PDF (jupyter nbconvert --to pdf) for easy sharing via email or LinkedIn
Alternative: If Jupyter feels like a stretch, even a clean Excel output with a summary tab and one chart is shareable. Don't let perfect be the enemy of published.
05
Publish, Share & Get Seen
Est. 1–2 days · Maximum visibility
FINISH LINE
  • Add the GitHub repo link to your LinkedIn profile under Featured or in the About section
  • Write a LinkedIn post announcing the project — include 1 interesting finding (e.g., "The optimizer consistently undervalued stretch bigs across all 3 cap scenarios")
  • Post on r/nba or r/nbadiscussion with your findings — sports fans engage heavily with this kind of analysis
  • Submit to Kaggle Notebooks as a public notebook for discoverability in the data science community
  • Add project to your resume under "Projects" with 2 bullet points: what you built + what tool/technique you used + one result
  • If you want a web app: deploy a simple Streamlit dashboard to Streamlit Community Cloud (free, no backend needed)
    Advanced — tackle after core portfolio is live
Talking point to prepare: "I built an NBA roster optimizer that uses K-Means clustering and mixed-integer linear programming to maximize team value across three salary cap scenarios — from scratch, with no prior coding experience." That's a compelling 30-second pitch.
